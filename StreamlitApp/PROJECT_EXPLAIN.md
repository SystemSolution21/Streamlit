# StreamlitAppプロジェクト

本ドキュメントは、プロジェクトの 仕様書（要件定義）＋設計書＋開発工程まとめ を統合した、全体像を示す技術文書である。データ可視化、株式市場分析、および LLM（大規模言語モデル）を統合したマルチページ型 Streamlit アプリケーションの構築における要件、設計、実装、テスト、運用までの流れを体系的に整理する。

## 1. 仕様書（要件定義）

### 1.1 プロジェクト概要

本プロジェクトは、データサイエンスと AI 機能を統合した高度なダッシュボード基盤を Streamlit で構築することを目的とする。株式データのリアルタイム分析、Plotly を用いた視覚化、および OpenAI やローカル LLM（Ollama）を活用した感情分析・チャット機能を一元的に提供する。

### 1.2 機能要件

- **株式市場ダッシュボード**: S&P 500 企業のデータ取得、価格トレンド、出来高、相関関係の視覚化。
- **AI 感情分析**: ニュース記事を解析し、ポジティブ/ネガティブ判定および将来予測の有無を抽出（OpenAI/Ollama 対応）。
- **LLM チャットボット機能**: 基本的なチャット、ファイルベースの Q&A（Anthropic）、検索エンジン連携（LangChain）。
- **データ管理**: CSV/Excel ファイルのアップロード支援および、解析結果の再エクスポート機能。
- **チュートリアル・サンプル**: 高度なフィルタリングやプロット手法のデモンストレーション。

### 1.3 非機能要件

- **インタラクティブ性**: ユーザー操作に対して Plotly 等を用いた即時的なフィードバックを行うこと。
- **拡張性**: `pages/` ディレクトリ構造を利用し、新しい分析ツールや LLM 例を容易に追加できること。
- **セキュリティ**: API キーをサイドバー経由で安全に入力・管理できる構造にすること。
- **保守性**: Poetry による依存関係管理と、共通ロジックの `utils` 化によるコードの再利用性を高めること。

### 1.4 想定ユーザー

- データアナリストおよび金融トレーダー
- AI アプリケーションの開発学習者
- データの可視化と AI 解析を組み合わせて試行したいユーザー

## 2. 設計書

### 2.1 アーキテクチャ設計

本プロジェクトは、Streamlit のマルチページ機能を活用したシングルページアプリケーション (SPA) 風の構成をとる：

- **Frontend/Backend**: Streamlit フレームワークにより統合管理。
- **LLM 連携層**: クラウド API (OpenAI, Anthropic) とローカル環境 (Ollama) のハイブリッド構成。
- **データソース**: Wikipedia (S&P 500 リスト), Stooq (株価データ), Google News (ニュースフィード)。

### 2.2 ディレクトリ構造

```text
StreamlitApp/
├── app.py              # メインエントリポイント
├── pages/              # マルチページ構成（Stocks, llm-examples, tutorial等）
├── utils/              # 共通ユーティリティ（データ処理、ヘルパー関数）
├── data/               # データセット保存用
├── docs/               # 技術ドキュメント、スタイルガイド
├── tests/              # Pytest によるテストスイート
├── pyproject.toml      # Poetry 依存関係管理
└── README.md           # プロジェクト全体説明書
```

### 2.3 モデル設計

- **データ構造化 (Pydantic)**: `FinancialSentimentAnalysis` クラスにより、LLM の出力を `sentiment` (極性) と `future_looking` (将来予測の有無) として厳密に定義。
- **状態管理 (st.session_state)**: チャット履歴、ユーザー入力、アップロードされた一時データの永続化。
- **プロンプトモデル**: 金融アナリストの役割を定義したシステムプロンプトの設計。

### 2.4 初期データ・外部 API 連携設計

- **株価データ**: Stooq API を介した 5 年分の日次データの取得。
- **市場構成**: Wikipedia からの S&P 500 銘柄リスト（シンボル、社名）の動的取得。
- **ニュースソース**: GNews を用いた最新の金融ヘッドラインの取得。

### 2.5 API / アプリ設計

- **Stocks アプリ**: 可視化、統計解析、相関分析、トレンド分析の提供。
- **LLM アプリ**: LangChain を用いた Web 検索連携、ファイルベースの Q&A。
- **Sentiment AI**: OpenAI (クラウド) と Ollama (ローカル) の選択可能な解析エンジン。

## 3. 開発工程まとめ（要件定義 → 設計 → 実装 → テスト → 運用）

### 3.1 要件定義工程

- 市場データ分析と AI 連携による投資判断補助の要件整理。
- マルチページ構成による機能拡張性の確保。
- ローカル LLM 利用によるプライバシー配慮の定義。

### 3.2 設計工程

- インタラクティブな UI コンポーネント（Tabs, Columns, Metrics）の配置設計。
- Pydantic を用いた LLM 出力フォーマットの設計。
- カスタム CSS によるダッシュボードのスタイリング設計。

### 3.3 実装工程

- `stocks.py` での Plotly グラフィック実装。
- OpenAI/Anthropic API および Ollama (gemma3) の統合。
- CSV/Excel アップロードおよびデータエクスポート機能の実装。
- 共通処理の `utils` モジュール化。

### 3.4 テスト工程

- Pytest を用いた S&P 500 シンボル取得ロジックの検証。
- 株価データ取得 API の接続テスト。
- 各種 LLM モデルからの構造化出力のパーステスト。

### 3.5 運用工程

- Poetry による依存パッケージの厳密な管理。
- Streamlit Community Cloud へのデプロイ環境の構築。
- `st.secrets` による API キー情報の秘匿管理。
